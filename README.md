# GitWorkflowTest
Test Git workflow 

### 워크플로우

<aside>

01 브랜치 생성 → 02 코드 작업 → 03 PR 요청 → 04 코드 리뷰 → 05 테스트 자동 실행 (CI)→ 06 승인 → 07 main 브랜치 머지

</aside>

**01 브랜치 전략** 

- main 브랜치에 직접 push 금지
- 브랜치 네이밍 : `feature/기능명`, `fix/버그명`, `hotfix/긴급수정명`, `docs/문서명`  , `exp/실험명`
- 머지 완료 후 브랜치 삭제

**02 커밋 규칙** 

- 형식 : {브랜치 이름} : 설명 ( 예시 : `feature(기능명): 설명`)
- 하나의 커밋 = 하나의 작업 단위

**03 PR규칙** 

- 하나의 PR = 하나의 기능/이슈
- 변경 범위 400줄 이하 권장
- 가능하다면 다른 1인이 승인 후 머지
- 테스트 통과 필수 (그러나 테스트가 어떤 식으로 진행되는지는 구체화 필요)
